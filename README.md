# Транспортный справочник

Система управления каталогом общественного транспорта с поддержкой маршрутизации, рендеринга карт и обработки JSON-запросов. Проект демонстрирует навыки работы с алгоритмами на графах, геометрическими вычислениями и объектно-ориентированным проектированием на C++17.

## Возможности

*   **Транспортный каталог:** Хранение данных об остановках и автобусных маршрутах (линейные и кольцевые). Поддержка реальных дорожных расстояний между остановками.
*   **Статистика:** Расчет детальной статистики по маршрутам: общее количество остановок, уникальные остановки, длина маршрута и коэффициент извилистости.
*   **Маршрутизация:** Поиск оптимального пути между двумя остановками с учетом времени ожидания и скорости движения. Использование алгоритма Флойда-Уоршелла для предвычисления кратчайших путей во взвешенном графе.
*   **Рендеринг карт:** Генерация SVG-изображений карты транспортной сети. Автоматическая проекция географических координат на плоскость, отрисовка маршрутов разными цветами, отображение подписей с подложкой для читаемости.
*   **JSON:** Полная поддержка формата JSON для входных данных (база, настройки) и выходных ответов. Собственная реализация парсера и билдера без внешних зависимостей.
*   **Безопасность и надежность:**
    *   Обработка ошибок ввода: некорректные имена остановок/автобусов, отсутствие маршрута.
    *   Защита от переполнения при парсинге чисел.
    *   Корректная работа с nullptr и пустыми контейнерами.

## Технологии и стек

*   **Язык:** C++17 (Standard Library, Variants, Optional, Smart Pointers, String View).
*   **Сборка:** Компилятор g++/clang++ с поддержкой C++17.
*   **Алгоритмы:** Флойда-Уоршелл (маршрутизация), Haversine formula (геодезические расстояния), BFS (обновление зависимостей в графе).
*   **Паттерны:** Builder (JSON), Strategy (SVG Object), Facade (RequestHandler), Template Method (RenderContext).

## Структура проекта

Проект модульный и разделен по зонам ответственности:

| Файл | Описание |
| :--- | :--- |
| `main.cpp` | Точка входа. Оркестрация загрузки данных, инициализации роутера и обработки запросов. |
| `json.h/cpp` | Библиотека для парсинга и сериализации JSON. Поддержка всех примитивных типов, массивов и словарей. |
| `json_builder.h/cpp` | Конструктор (Builder) для программного создания JSON-структур с контролем контекста. |
| `json_reader.h/cpp` | Связующее звено: читает входной JSON, наполняет каталог, вызывает обработчики запросов. |
| `domain.h/cpp` | Базовые структуры данных (`Stop`, `Bus`, `BusStat`). |
| `transport_catalogue.h/cpp` | Основной класс хранения данных. Индексы для быстрого поиска, расчет статистики маршрутов. |
| `geo.h/cpp` | Работа с географическими координатами. Расчет расстояний по формуле гаверсинусов. |
| `graph.h` | Шаблонный класс направленного взвешенного графа с списками смежности. |
| `router.h` | Реализация алгоритма поиска кратчайших путей (Флойда-Уоршелл) на графе. |
| `transport_router.h/cpp` | Адаптер транспортного каталога к графу. Построение ребер с весами (время в пути). |
| `request_handler.h/cpp` | Фасад для доступа к функционалу каталога и роутера. Унифицированный интерфейс для запросов. |
| `svg.h/cpp` | Библиотека для генерации SVG-документов. Абстрактные объекты, контекст рендеринга, цвета. |
| `map_renderer.h/cpp` | Логика отрисовки карты транспорта в SVG. Сферический проектор, настройка стилей. |
| `ranges.h` | Вспомогательная утилита для работы с диапазонами (Range-based for over containers). |

## Запуск

Приложение читает входные данные из `stdin` и пишет результат в `stdout`.

```bash
./transport_catalogue < input.json > output.json
```

## Архитектурные решения

1.  **Разделение модели и представления:** `TransportCatalogue` хранит только бизнес-данные, `MapRenderer` отвечает исключительно за визуализацию. Это позволяет менять формат вывода (например, на PNG или Web) без изменения ядра.
2.  **Предвычисление маршрутов:** Алгоритм Флойда-Уоршелла запускается один раз при инициализации `TransportRouter`. Последующие запросы маршрутов выполняются за O(1) по времени (без учета восстановления пути), что критично для интерактивных систем.
3.  **Гибкое управление стилями:** Все параметры рендеринга вынесены в структуру `RenderSettings` и загружаются из JSON. Можно динамически менять цвета, шрифты и размеры без перекомпиляции.
4.  **Типобезопасный JSON:** Парсер использует `std::variant` и строгую типизацию. Ошибки парсинга выбрасываются как исключения `ParsingError` с точным указанием места проблемы.
5.  **Эффективное хранение графа:** Используется вектор списков смежности (`incidence_lists_`) вместо матрицы смежности, что экономит память на разреженных графах транспортных сетей.
